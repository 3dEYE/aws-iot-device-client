# Repository guidance for coding agents

## Codex Cloud environment

- Run build and test commands from the repository root.
- During the agent phase, use the prepared `build`, `out/armhf`, and
  `out/aarch64` build trees. Do not install or download dependencies, and do
  not run `apt`, `apt-get`, `curl`, `wget`, `git fetch`, or `git clone`.
- Do not delete or reconfigure the prepared build trees with `cmake -S` or
  `cmake -B` during routine offline work.
- If a prepared build tree or toolchain is missing, report an environment setup
  failure instead of downloading dependencies.
- If `cmake --build` triggers regeneration that cannot complete offline, stop
  and report that the cached environment must be refreshed.

## Generated version file

- `.version` is generated by CMake.
- Never stage or commit `.version`.
- At the start of a task, check `git status --short -- .version`. If it already
  contains a user change, preserve that change instead of overwriting it.
- In a clean checkout, protect builds with:

  ```bash
  trap 'git restore --source=HEAD --worktree -- .version' EXIT
  ```

- Set the trap in the same shell invocation that runs the build commands. A trap
  created by an earlier, separate shell command provides no protection.
- Restore `.version` before reporting final Git status, even after a failed
  build.

## Native build and tests

Build the application and unit-test executable:

```bash
cmake --build build \
  --target aws-iot-device-client test-aws-iot-device-client \
  --parallel 2
```

Run the native unit tests:

```bash
AWS_CRT_MEMORY_TRACING=1 \
  ./build/test/test-aws-iot-device-client --gtest_brief=1
```

## ARM cross-builds

ARM32 means ARMv7 hard-float (`armhf`) in this repository. Build both the
application and test executable:

```bash
cmake --build out/armhf \
  --target aws-iot-device-client test-aws-iot-device-client \
  --parallel 2

cmake --build out/aarch64 \
  --target aws-iot-device-client test-aws-iot-device-client \
  --parallel 2
```

Do not execute the cross-compiled test binaries on the x86-64 Cloud host. Verify
their architectures instead:

```bash
arm-linux-gnueabihf-readelf -h out/armhf/aws-iot-device-client
arm-linux-gnueabihf-readelf -h \
  out/armhf/test/test-aws-iot-device-client

aarch64-linux-gnu-readelf -h out/aarch64/aws-iot-device-client
aarch64-linux-gnu-readelf -h \
  out/aarch64/test/test-aws-iot-device-client
```

Expected results:

- ARMHF application and tests: `Class: ELF32`, `Machine: ARM`.
- AArch64 application and tests: `Class: ELF64`, `Machine: AArch64`.

Finish validation with:

```bash
git restore --source=HEAD --worktree -- .version
git status --short
git diff --check
```
